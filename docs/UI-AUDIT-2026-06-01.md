# LevelFITness UI/UX Audit Report (2026-06-01)

## Executive Summary

This audit reviews the LevelFITness UI across **web and mobile** platforms against [Web Interface Guidelines](https://github.com/vercel-labs/web-interface-guidelines/main/command.md), with specific focus on:
- **Landing → Login/Signup → Dashboard flows**
- **Coach dashboard options & design patterns**
- **Client dashboard options & design patterns**
- **Calendar & workout views**
- **Logout experience & discoverability**
- **Readability & accessibility**
- **3D design opportunities**

---

## 🔴 CRITICAL ISSUES (Ship-Blocking)

### 1. **Navbar Logo Link Missing Home Navigation**
**File:** `frontend/components/landing/navbar.tsx`

The logo in navbar is not wrapped in a Link/navigation element. Users cannot click logo to go home.

```
navbar.tsx:39 - Logo should navigate to homepage on click
```

**Fix:** Make logo clickable and linked to homepage.

---

### 2. **Logout Button Insufficient Visual Hierarchy**
**File:** `frontend/components/dashboard/dashboard-sidebar.tsx:118`

The "Sign out" button is low-contrast (`text-bone-fade`) and positioned at bottom of sidebar. Mobile users may miss it. Should be:
- Higher contrast (at least `text-bone` normal)
- Better positioned or styled
- Clear confirmation required on mobile

```
dashboard-sidebar.tsx:118 - Sign out button low contrast (bone-fade), difficult to find
dashboard-sidebar.tsx:118 - Missing confirmation dialog before logout
```

**Recommendation:** 
- Move to prominent position or add confirmation modal
- Use `text-bone` minimum contrast
- Add icon (LogOut) - already done ✓

---

### 3. **Calendar Not Looking Like Real Calendar**
**File:** `frontend/components/workout/training-calendar.tsx:1-150`

Current implementation is a basic grid without:
- Proper day-of-week headers (Mon/Tue style)
- Today highlighting
- Week number badge
- Visual "grid lines" separating cells
- Touch-friendly cell sizing (current may be <44px)

```
training-calendar.tsx:8 - Calendar cells may be too small for touch (44px minimum)
training-calendar.tsx:120+ - Missing visual week structure (no real grid borders)
training-calendar.tsx:120+ - Today highlighting not visible
```

**Recommendation:**
- Add visual grid borders (border-grid style)
- Highlight today with a distinct background color
- Ensure cells are at least 44×44px for touch
- Add subtle alternating row colors for week grouping

---

### 4. **Auth Pages Missing Color Scheme Meta Tag**
**File:** `frontend/app/layout.tsx:6`

The root layout lacks `<meta name="theme-color">`. Dark mode inputs on Windows may render incorrectly.

```
layout.tsx:6 - Missing <meta name="theme-color"> for dark mode consistency
```

**Fix:** Add to root layout:
```html
<meta name="theme-color" content="#05060a" />
<meta name="color-scheme" content="dark light" />
```

### 5. **Focus Indicators Missing on Navigation Links**
**File:** `frontend/components/landing/navbar.tsx:34-37`

Navigation links lack visible focus indicators. Keyboard users cannot see focus state.

```
navbar.tsx:34 - Navigation links missing focus-visible ring
navbar.tsx:49 - Mobile menu links missing focus-visible ring
```

**File:** `frontend/app/(auth)/login/page.tsx`, `frontend/app/(auth)/signup/page.tsx`

```
login/page.tsx:48 - Email input has autocomplete=email ✓

signup/page.tsx - Same issues across all auth fields
```

---

### 8. **Mobile Viewport Not Set Correctly**
**File:** `frontend/app/layout.tsx:11`

Viewport is set but missing key mobile attributes:

```
layout.tsx:11 - Missing user-scalable, initial-scale should not be locked
```

**Current:**
```tsx
export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
};
```

**Should be:**
```tsx
export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  userScalable: true,
  minimumScale: 1,
  maximumScale: 5,
};
```

### 9. **Dashboard Button Variant "outline" Not Defined**
**File:** `frontend/components/ui/button.tsx`, `frontend/app/(dashboard)/coach/risk-signals/page.tsx`

Some pages use `variant="outline"` but it's not defined in Button component.

```
risk-signals/page.tsx - Uses variant="outline" which doesn't exist
```

```tsx
outline: {

  focus: 'focus-visible:ring-primary/30',
}
```

## 🟠 DESIGN & UX GAPS

### 11. **Coach Dashboard Missing Clear Option Grouping**
**File:** `frontend/components/dashboard/dashboard-sidebar.tsx` (Coach links)

Coach links are 16 items flat list. Should group by function:
- **Coaching** (Command center, Recovery, Intelligence, Risk signals)
- **Content** (Workouts, Programs, Tasks)
- **Management** (Clients, Packages, Billing)
- **Insights** (Analytics, Client health, Feed)

**Recommendation:**
```tsx
const coachLinks = {
  coaching: [
    { label: 'Command center', ... },
    { label: 'Recovery', ... },
    { label: 'Intelligence', ... },
    { label: 'Risk signals', ... },
  ],
  content: [ ... ],
  // etc
};
```

Render with section headers:
```tsx
<div className="space-y-6">
  {Object.entries(coachLinks).map(([section, items]) => (
    <div key={section}>
      <h3 className="px-3 py-2 text-xs font-bold text-muted-foreground uppercase tracking-wide">{section}</h3>
      {linkList(items, pathname)}
    </div>
  ))}
</div>
```

### 13. **Calendar Missing Mobile Responsiveness**
**File:** `frontend/components/workout/training-calendar.tsx`

Currently designed for desktop. On mobile:
- Week view becomes cramped
- Day labels cut off
- Workout cards stack weirdly
- No horizontal scroll

**Recommendation:**
- Add responsive grid: `grid-cols-7 md:grid-cols-7` (always 7 cols)
- Add horizontal scroll on mobile: `overflow-x-auto`
- Reduce day label size on mobile: `text-xs md:text-sm`
- Stack workout cards vertically on mobile with horizontal scroll

## 📊 READABILITY & TYPOGRAPHY FIXES

### 21. **Form Labels Not Using Semantic HTML**
**File:** `frontend/app/(auth)/login/page.tsx:47`

Labels are correct (`<label htmlFor="...">`) ✓. But missing:
- `required` attribute on `<input>`
- `aria-required="true"` for assistive tech

```html
<input 
  id="login-email"
  required
  aria-required="true"
  aria-describedby="email-error"
/>
```

---

### 22. **Error Messages Using Ellipsis Wrong**
**File:** `frontend/components/landing/navbar.tsx` and elsewhere

Some places use `"..."` instead of proper ellipsis `"…"`.

**Audit result:**
```
Components/loading states missing "…" suffix
Example: "Loading" should be "Loading…"
```

---

### 23. **Numbers Not Using Tabular Numerals**
**File:** Throughout stats/metrics displays

When displaying:
- Session counts
- Streak numbers
- Client counts

Use `font-variant-numeric: tabular-nums` to align columns.

```css
.metric-number {
  font-variant-numeric: tabular-nums;
}
```

---

### 24. **Curly Quotes & Non-Breaking Spaces Missing**
**Throughout:** Copy uses straight quotes `"` instead of curly `"` `"`

Also: Prices, measurements, measurements should use non-breaking spaces.

Example fixes:
- `"Welcome back"` → `"Welcome back"` (curly quotes)
- `10 MB` → `10&nbsp;MB` (non-breaking)
- `⌘ K` → `⌘&nbsp;K`

## 🎨 DESIGN CONSISTENCY AUDIT

### 25. **Color Token Usage Inconsistent**
**Files:** Multiple

Some components use hardcoded colors vs. semantic tokens:
- `text-bone-mute` (correct) ✓
- `text-primary` (correct) ✓
- But missing in some places: hover states use `hover:text-bone` instead of `hover:text-primary`

---

### 26. **Border Radius Inconsistent**
**Files:** Multiple

Mix of:
- `rounded-lg` (login button)
- `rounded-2xl` (cards)
- `rounded-xl` (inputs)
- `rounded-3xl` (large cards)

**Standardize:**
- Input/button: `rounded-lg`
- Card: `rounded-2xl`
- Large sections: `rounded-3xl`

---

### 27. **Shadow & Depth System Underutilized**
**Current:** Boxes are flat with thin borders. **Add:**
- `shadow-sm` on hover for interactive elements
- `shadow-md` on cards
- `shadow-lg` on modals

## 📱 MOBILE-SPECIFIC FINDINGS

### Issues
1. Hamburger menu closing on nav (✓ works) but needs focus trap
2. Calendar grid too narrow for touch
3. Stats cards stack poorly on small screens
4. Quick actions cards overflow on mobile

### Recommendations
1. **Bottom sheet for mobile nav** — Slides up from bottom instead of left (better thumb reach)
2. **Touch-friendly calendar** — Single week view on mobile, swipe to next week
3. **Card deck layout** — Use horizontal scroll for grouped cards instead of vertical stack
4. **Larger touch targets** — Enforce 44×44px minimum
